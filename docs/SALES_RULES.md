# Overseas Sales Rules

## 适用范围

本文件约束 LYGD AI Tunnel Platform 中海外客户和项目商机的基础数据管理。CRM 当前只保存文件型数据，不接 UI、不调用外部 API，也不自动发送消息或报价。

## 客户与国家

1. 每个海外客户必须有唯一 customerId 和 companyName。
2. 客户 countryCode 必须引用国家知识库中的国家代码。
3. 联系方式属于敏感数据，只能保存业务所需信息；公开示例不得包含真实个人信息。
4. 同一客户的历史项目通过 historicalProjectIds 关联，禁止用客户名称代替稳定编号。

## 商机阶段

受控销售阶段如下：

1. `lead`：获得线索，尚未建立有效沟通。
2. `contacted`：已与客户建立初步联系。
3. `technical_discussion`：正在确认隧道与台车技术需求。
4. `quotation_sent`：已发送经过内部审核的报价。
5. `negotiation`：正在进行技术或商务谈判。
6. `won`：商机已确认赢单。
7. `lost`：商机已确认失败。

阶段变更必须由授权人员记录。需要跳过常规阶段时，应在业务记录中保留原因；系统不得自动推断 won 或 lost。

## 预计金额

- 预计金额不等于正式报价、合同金额或收入确认。
- 没有可靠依据时，amount 必须为 `null`，不得虚构。
- 必须记录币种和核实状态。
- 正式报价仍遵循 Quotation Engine 和内部审批规则。

## 技术与重量边界

- 隧道类型和台车需求必须来自客户资料或技术沟通。
- 经验重量必须标明“历史经验参考，不替代正式设计”。
- 水工隧洞重量必须人工确认。
- 商机数据不得直接修改 Weight Engine 或 Quotation Engine 规则。

## 国家与运输

- 海外项目必须引用国家知识库。
- 语言、币种和运输备注必须有可说明来源。
- 未确认的运输要求使用空值或空数组，不得自行补齐。
- 运输备注不构成物流承诺，正式运输方案必须复核。

## 下一步行动

- 活跃商机应记录下一步行动、计划日期和责任人。
- 信息尚未确定时使用 `null`，不得虚构人员或日期。
- 下一步行动仅用于内部跟进，不触发外部通知。

## 数据与隐私

- 不得在公开示例、测试日志或提交信息中写入真实联系方式、合同价格或敏感客户信息。
- 数据修改应保留版本并接受人工审核。
- 本模块不接外部 CRM、邮件、即时通讯或第三方数据服务。
