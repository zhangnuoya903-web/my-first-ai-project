# Product Roadmap

本路线图表达产品演进方向，不代表未开始 Sprint 已获开发授权。

## 阶段一：可运行原型

- V2.0：形成隧道二衬台车报价页面。
- Sprint 1：修复 DOM 控件、计算和重置行为。
- Sprint 2：完成 JavaScript ES Modules 模块化。

状态：已完成。

## 阶段二：项目治理

- Sprint 0：建立项目章程、产品愿景、业务规则、系统架构、Sprint 计划、变更日志和 AI 开发约束。
- 建立人工审核后再进入下一 Sprint 的门禁。

状态：当前完成，等待审核。

## 阶段三：规则与数据治理

计划方向：

- 将已批准的重量经验数据与业务代码分离。
- 建立数据来源、版本、生效日期和维护责任。
- 为价格数据建立独立存储和审批边界。
- 为人工修正保留原因与审计信息。

状态：未开始。

## 阶段四：报价生命周期

计划方向：

- 项目、客户及报价记录持久化。
- 报价版本、复制、查询和审批。
- 报价明细与规则版本快照。
- 受控的打印或文档导出。

状态：未开始。

## 阶段五：平台化

计划方向：

- 权限和角色管理。
- 数据审计、备份与恢复。
- 自动化测试、持续集成和受控部署。
- 多项目、多地区和多币种能力评估。

状态：未开始。

## 路线图约束

任何阶段都不得绕过工程师确认，不得把经验重量升级为设计重量，也不得在缺少业务批准时生成价格规则。
